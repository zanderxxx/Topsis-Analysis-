# L99 产品介绍页 TOPSIS 关注度分析

对产品介绍页用户行为数据执行完整的 TOPSIS（逼近理想解排序法）关注度分析流水线。从 Excel 数据文件出发，经过数据清洗、异常修正、聚合计算、TOPSIS 评分，最终输出结构化的排名报告。

## 适用场景

- 用户有产品介绍页的用户行为数据（Excel 格式）
- 数据包含：每一屏名称、曝光时长、曝光人数、曝光次数、退出人数、日期等字段
- 需要对各屏幕进行关注度排名分析

## 工作流程

### Step 1: 确认输入文件

找到用户指定的 Excel 文件路径。文件应包含以下列：
- `每一屏名称` — 屏幕标识
- `每一屏的曝光时长` — 曝光时长（数值）
- `每一屏的曝光人数` — 曝光人数（数值）
- `每一屏的曝光次数` — 曝光次数（数值）
- `最后一屏的退出人数` — 退出人数（数值）
- 日期列（列名包含"日期"或"date"）

如果用户没有明确指定文件路径，在桌面或当前目录搜索包含"介绍页"、"用户行为"等关键词的 Excel 文件。

### Step 2: 确认输出位置

默认将结果输出到与输入文件相同的目录，文件名为 `L99_TOPSIS_Analysis_Result.xlsx`。如果用户有特殊要求则按其指定。

### Step 3: 确保项目环境就绪

本项目使用 Python 虚拟环境。进入项目根目录后：

1. 检查虚拟环境是否存在：
```bash
ls .venv/bin/python
```

2. 如果虚拟环境不存在，创建它：
```bash
python3 -m venv .venv
```

3. 安装依赖：
```bash
.venv/bin/pip install -r requirements.txt
```

### Step 4: 运行分析

使用命令行脚本执行分析：

```bash
.venv/bin/python scripts/run_analysis.py "<输入文件绝对路径>" "<输出文件绝对路径>"
```

如果只提供输入文件路径，结果默认输出到输入文件同目录下：
```bash
.venv/bin/python scripts/run_analysis.py "<输入文件绝对路径>"
```

### Step 5: 汇报结果

运行完成后：
1. 告知用户输出文件的位置
2. 展示 TOPSIS 关注度 Top 10 排名
3. 简要说明结果含义：
   - `TOPSIS关注度得分`：0~1，越高表示该屏幕越受关注
   - `关注度排名`：综合排名
   - `平均曝光时长`：正向指标（权重 0.7）
   - `退出率`：负向指标（权重 0.3）

## 配置参数说明

以下参数在 `app/config.py` 中配置，可按需调整：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `EXPOSURE_THRESHOLD` | 20 | 曝光人数低于此值的数据会被剔除 |
| `AVG_DURATION_ANOMALY_THRESHOLD` | 1000 | 日均曝光时长超过此值视为异常 |
| `EXCLUDE_SCREENS` | `["scene-download", "scene-overview-firstscreen scene-first"]` | 需要排除的特殊页面 |
| `TOPSIS_WEIGHTS` | `{"平均曝光时长": 0.7, "退出率": 0.3}` | TOPSIS 各指标权重 |

## 分析流水线详解

1. **数据清洗**（app/cleaning.py）
   - 剔除曝光人数 < 20 的低质量数据
   - 排除下载页、首屏等特殊页面
   - 异常值修正：日均曝光时长 > 1000 的记录，用前后日期的均值替代

2. **聚合计算**（app/aggregation.py）
   - 按屏幕名称聚合所有日期数据
   - 计算平均曝光时长 = 总曝光时长 / 总曝光人数
   - 计算退出率 = 退出人数 / 曝光人数

3. **TOPSIS 分析**（app/topsis.py）
   - MinMaxScaler 归一化
   - 退出率取反（负向指标）
   - 加权计算（平均曝光时长 0.7，退出率 0.3）
   - 计算到正负理想解的欧氏距离
   - 计算贴近度得分并排名

4. **报告导出**（app/report.py）
   - Sheet 1：TOPSIS 关注度排名（完整结果）
   - Sheet 2：异常修正日志（记录所有被修正的数据点）
